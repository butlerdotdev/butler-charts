Butler Provider Proxmox has been deployed!

This provider enables Butler to provision VMs on Proxmox VE.

To check the provider status:
  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "butler-provider-proxmox.name" . }}

To view provider logs:
  kubectl logs -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "butler-provider-proxmox.name" . }} -f

Prerequisites for Proxmox:
  1. Upload Talos image to Proxmox (with qemu-guest-agent extension)
  2. Create a VM network in Proxmox
  3. Create a ProviderConfig with Proxmox kubeconfig

Example ProviderConfig:
  apiVersion: butler.butlerlabs.dev/v1alpha1
  kind: ProviderConfig
  metadata:
    name: proxmox-provider
    namespace: butler-system
  spec:
    provider: proxmox
    credentialsRef:
      name: proxmox-kubeconfig
      namespace: butler-system
      key: kubeconfig
    proxmox:
      namespace: default
      networkName: default/vm-network
      imageName: default/talos-v1.9.0

Documentation: https://docs.butlerlabs.dev/providers/proxmox
